1. システム概要
目的

ふれあい太田職員が、勤務表を見る感覚で予定を確認・編集できるシステムを構築する。

複雑なグループウェアではなく、

「見やすい・速い・簡単」

を最優先とする。

スマホ対応
一つだけ追加したい項目

仕様書とは別に、GitHubに CHANGELOG.md を置こう。

例えば、

v0.1.0
・初期画面作成

v0.2.0
・カレンダー表示追加

v0.3.0
・編集画面追加

こんな感じで更新履歴を残していく。

この仕様書を「Version 1.0の設計書」として固定しよう。

今後は、この仕様書を基準に開発を進めて、思いついた新機能は仕様書を変えずにアイデアリストへ追加する運用にすれば、途中で迷子にならずに完成まで進められると思う。
